下面详细解释 `Qwen3VLMoeModel.forward` 中每个输入参数的含义、用途及获取方式。这些参数涵盖了多模态大模型（文本 + 图像/视频）的前向所需的所有信息。
## 1. `input_ids` (torch.LongTensor, optional)

- **形状**：`(batch_size, seq_len)`
    
- **含义**：文本 token 的整数 ID 序列。每个整数对应词表中的某个 token。
    
- **用途**：模型通过嵌入层（`get_input_embeddings()`）将其转换为 `inputs_embeds`。如果直接提供了 `inputs_embeds`，则 `input_ids` 可以被忽略。
    
- **如何获得**：
    
    - 训练/推理时，通过 tokenizer 对原始文本进行编码得到。
        
    - 在多模态数据中，文本中会包含特殊的图像/视频占位符 token（如 `<|vision_start|>`, `<|image_pad|>` 等），这些 token 的位置会被后续视觉 token 替换。
        
    - 在序列并行（SP）下，`input_ids` 会在 `SequenceParallelCollator` 中被切分成 `seq_len // sp_size` 长度的块，每个 rank 只得到一部分连续 token。
        

---

## 2. `attention_mask` (torch.Tensor, optional)

- **形状**：`(batch_size, seq_len)`
    
- **含义**：注意力掩码，`1` 表示有效 token，`0` 表示填充 token（padding）。用于告诉模型哪些位置不应该被注意力机制关注。
    
- **用途**：在标准 Transformer 中用于屏蔽填充位置。但在使用 Flash Attention 且传入 `cu_seqlens` 时，该 mask 可能被忽略或强制设为全 `1`。
    
- **如何获得**：
    
    - 由 tokenizer 返回，或在 collator 中根据序列长度生成。
        
    - 在 VeOmni 的 SP 模式下，`attention_mask` 被强制填充为 `1`，因为 Flash Attention 使用 `cu_seqlens` 和 `position_ids` 处理变长序列，不再依赖原始 mask。
        

---

## 3. `position_ids` (torch.LongTensor, optional)

- **形状**：`(batch_size, seq_len)` 或 `(3, batch_size, seq_len)`（当使用 3D RoPE 时）
    
- **含义**：每个 token 的位置索引，用于旋转位置编码（RoPE）。对于多模态，图像/视频 token 的位置需要根据它们在空间网格中的坐标重新计算。
    
- **用途**：在计算 Q/K 的 RoPE 时提供位置信息。正确的 position_ids 是正确建模多模态序列顺序的关键。
    
- **如何获得**：
    
    - 在 `Qwen3VLMoeModel.forward` 中，如果未提供，会调用 `self.get_rope_index()` 根据 `input_ids`、`image_grid_thw`、`video_grid_thw` 和 `attention_mask` 动态计算。
        
    - 计算时会考虑每张图像/视频产生的 token 数量（`grid_thw` 的乘积除以 `spatial_merge_size²`），然后为视觉 token 分配连续的位置索引，文本 token 的位置则依次递推。
        
    - 在 SP 模式下，`position_ids` 也会被切分，但注意其切分方式需要与 `input_ids` 保持一致（通常先计算全局 `position_ids`，再按 rank 切片）。
        

---

## 4. `past_key_values` (Optional[Cache])

- **形状**：是一个缓存对象，通常是 `DynamicCache` 或类似结构，存储了之前所有层的 K 和 V 张量。
    
- **含义**：用于推理时的 KV 缓存，避免重复计算已经处理过的 token。
    
- **用途**：在自回归生成时，每生成一个新 token，只需要计算该 token 的 Q/K/V，并与缓存的 K/V 拼接后做注意力。
    
- **如何获得**：
    
    - 由模型在上一次前向调用时返回，并保存下来。
        
    - 首次前向时该值为 `None`。
        

---

## 5. `inputs_embeds` (torch.FloatTensor, optional)

- **形状**：`(batch_size, seq_len, hidden_size)`
    
- **含义**：直接输入的 token 嵌入向量，而不是通过 `input_ids` 查表得到。
    
- **用途**：当需要绕过嵌入层（例如已经预先计算好了嵌入，或者需要替换视觉 token 的嵌入）时使用。
    
- **如何获得**：
    
    - 若未提供，模型会使用 `self.get_input_embeddings()(input_ids)` 生成。
        
    - 在多模态处理中，会先用文本嵌入初始化 `inputs_embeds`，然后通过 `masked_scatter` 将图像/视频的嵌入替换到对应位置。
        

---

## 6. `labels` (torch.LongTensor, optional)

- **形状**：`(batch_size, seq_len)`
    
- **含义**：用于计算损失的目标 token ID。对于因果语言建模，通常是将 `input_ids` 向右移位一位（第一个 token 忽略，最后一个 token 预测下一个）。
    
- **用途**：计算交叉熵损失。值为 `IGNORE_INDEX`（通常为 -100）的位置不参与损失计算。
    
- **如何获得**：
    
    - 在训练时，由数据加载器提供。通常在 collator 中进行移位操作（`labels = input_ids[..., 1:]` 并填充最后一位为 `IGNORE_INDEX`）。
        
    - 在多模态数据中，视觉 token 对应的 `labels` 通常设为 `IGNORE_INDEX`，因为不需要预测图像内容。
        

---

## 7. `pixel_values` (torch.Tensor, optional)

- **形状**：`(num_images, channels, height, width)`
    
- **含义**：原始图像的像素值。`channels` 通常为 3（RGB），`height` 和 `width` 是原始图像尺寸（或缩放后的尺寸）。
    
- **用途**：输入到视觉编码器（ViT）中，提取图像特征嵌入。
    
- **如何获得**：
    
    - 由数据加载器从图像文件读取，并经过预处理（缩放、归一化、转换为张量）。
        
    - 如果批次中没有图像，则为 `None`。
        

---

## 8. `pixel_values_videos` (torch.FloatTensor, optional)

- **形状**：`(num_videos, num_frames, channels, height, width)` 或类似，取决于实现。
    
- **含义**：视频的像素值。视频通常被采样为多帧图像。
    
- **用途**：输入到视觉编码器（支持时序的 ViT）中，提取视频特征。
    
- **如何获得**：类似 `pixel_values`，但需要从视频文件中抽取帧。
    

---

## 9. `image_grid_thw` (torch.LongTensor, optional)

- **形状**：`(num_images, 3)`，每行对应 `(temporal, height, width)`。
    
- **含义**：描述每张图像在视觉编码器输出的特征图上的网格尺寸。例如，输入图像经过 patch embedding 和 spatial merge 后，特征图的尺寸为 `(H // patch_size // merge_size, W // patch_size // merge_size)`。`temporal` 通常为 1（静止图像）。
    
- **用途**：
    
    - 计算每张图像产生的 token 数量：`temporal * height * width`。
        
    - 用于生成 `position_ids` 和 `cu_seqlens`，以便将视觉 token 正确插入序列。
        
- **如何获得**：
    
    - 由数据加载器或视觉编码器的预处理阶段根据原始图像大小和模型参数（patch size, merge size）计算得到。
        

---

## 10. `video_grid_thw` (torch.LongTensor, optional)

- **形状**：`(num_videos, 3)`
    
- **含义**：类似 `image_grid_thw`，但用于视频。`temporal` 维度表示视频的帧数（经过采样后）。
    
- **用途**：计算视频 token 数量，并用于位置编码。
    
- **如何获得**：同 `image_grid_thw`，但基于视频的帧数和分辨率。
    

---

## 11. `cache_position` (torch.LongTensor, optional)

- **形状**：`(seq_len,)` 或 `(1, seq_len)`
    
- **含义**：当前序列中每个 token 在整个生成过程中的绝对位置索引（从 0 开始）。用于推理时，结合 `past_key_values` 判断哪些 token 是新的。
    
- **用途**：帮助模型正确计算 RoPE 索引和缓存的位置偏移。
    
- **如何获得**：
    
    - 首次前向时通常为 `None`，模型内部会根据 `input_ids` 长度自动生成。
        
    - 生成下一个 token 时，`cache_position` 会被设为 `[past_length]`。
        

---

## 12. `logits_to_keep` (Union[int, torch.Tensor])

- **形状**：整数或标量张量。
    
- **含义**：控制模型返回多少个位置的 logits。例如设为 `1` 时只返回最后一个 token 的 logits。
    
- **用途**：减少显存占用和计算量，尤其在推理时只需要最后几个 token 的 logits 来采样下一个 token。
    
- **如何获得**：由调用者指定，通常是生成逻辑中的参数。
    

---

## 13. `**kwargs` (Unpack[TransformersKwargs])

- **包含的可能参数**：
    
    - `cu_seqlens_q` / `cu_seqlens_k`：Flash Attention 所需的累积序列长度，用于变长序列优化。
        
    - `max_length_q` / `max_length_k`：序列中的最大长度。
        
    - `visual_pos_masks`：指示哪些位置是视觉 token 的布尔掩码（用于 deepstack 特征融合）。
        
    - `deepstack_visual_embeds`：视觉编码器中间层输出的特征，与语言模型的 deepstack 机制结合。
        
    - `rope_deltas`：RoPE 的偏移量（用于多模态动态位置）。
        
- **如何获得**：主要由模型内部或上层调用者（如训练循环）根据需求设置。
    

---

## 这些参数如何从原始数据一步步产生？

以一次训练迭代为例：

1. **原始数据**：文本 + 图像文件路径。
    
2. **Tokenizer**：将文本转为 `input_ids`，并生成初始的 `attention_mask`（1 表示有效文本，0 表示填充）。
    
3. **图像预处理**：读取图像，缩放、归一化，得到 `pixel_values`；同时根据图像尺寸和模型配置计算 `image_grid_thw`。
    
4. **Collator**：
    
    - 将多个样本拼成一个 batch，对序列进行 padding 到相同长度。
        
    - 可选地，对 `labels` 进行移位（因果 LM）。
        
    - 如果启用 SP，调用 `SequenceParallelCollator` 对 `input_ids`、`labels`、`position_ids` 等沿序列维度切分。
        
5. **模型前向**：
    
    - 若未提供 `inputs_embeds`，则通过嵌入层得到初始嵌入。
        
    - 若有 `pixel_values`，调用视觉编码器得到 `image_embeds` 和 `deepstack_image_embeds`。
        
    - 通过 `masked_scatter` 将视觉嵌入替换到文本嵌入中对应位置（由图像占位符 token 指示）。
        
    - 生成 `position_ids`（若未提供）。
        
    - 将最终嵌入送入语言模型。
        

所有参数的设计都是为了在多模态、变长序列、分布式并行（SP/FSDP）等复杂场景下正确、高效地完成前向计算。
